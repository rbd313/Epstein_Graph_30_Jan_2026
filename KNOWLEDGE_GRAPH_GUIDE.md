# The Epstein Network Knowledge Graph: An Insurance Lens

A knowledge graph is a database of relationships. Instead of storing isolated facts ("Person A exists," "Person B exists"), it stores **connections** ("Person A met Person B; they both invested in Property C; this generated Contract D").

For insurance professionals, think of it as a **claims network map**: you don't just know that Claim X happened alone—you see it connected to similar claims, overlapping parties, shared risk factors, and hidden patterns that signal fraud, concentration, or systemic exposure.

Below are five interactive views of the Epstein network. Each reveals a different truth about how networks work—and how insurance should analyze them.

### The Insurance Principle: **Exposure Mapping**
In property insurance, you don't assess one building—you assess the portfolio. A claim on Building A becomes relevant if:
- It uses the same contractor as Building B (shared risk)
- It's in the same neighborhood fire district (concentration)
- The same property manager oversees both (systemic weakness)

**The Epstein graph shows exposure concentration:**

When one node in the cluster is compromised, the entire network becomes a liability.

## 🛡️ For Your Insurance Portfolio

### Questions to Ask Underwriters:

1. **Concentration**: "Are we exposed to a single hub? If yes, what's our exposure if that hub is compromised?"

2. **Cascade**: "If Node A is implicated, which other nodes in our portfolio face contagion?"

3. **Opacity**: "How many of our claims flow through non-transparent entities? Can we map the actual decision-makers?"

4. **Velocity**: "For high-velocity risk (entertainment, media, reputation), how quickly can we detect network changes?"

5. **Systemic**: "Do we have policies on nodes that are tightly clustered? If that cluster fails, what's our maximum loss?"

Insurance is fundamentally about **understanding risk**. Knowledge graphs turn networks into risk intelligence.
